# 影札（カゲフダ）

宣言ブラフ×カウンティングの対戦カードゲーム。全員が同じ「1〜7」の数字札を7枚だけ持ち、
嘘の宣言をつき通すか、ダウトで暴くかを競います。2〜5人用・全7ラウンド。

## 遊び方

1. 手札から1枚を伏せて選び、全員同時に「宣言値」を出す（嘘OK。3を出して「7」と言ってよい）
2. 宣言値が最大の人が暫定勝者（同値は先に宣言した方）
3. 他の全員に約20秒の「ダウト」チャンス（早い者勝ちで1人だけ）。「見送る」も選べて、全員が見送ればすぐ次へ
   - ダウト成立 → 札を公開。嘘なら宣言者 −1点・ダウト側 ＋2点。本当なら宣言者 ＋2点・ダウト側 −1点
   - 誰もダウトしなければ暫定勝者 ＋1点。札は公開されない
4. 使った札は戻らない。7ラウンド後、最多得点の勝ち
5. 同点のときは「嘘を見破った回数が多い方 → より遅いラウンドで得点した方」の順で上位。それでも並べば同着

## モード

- **オンライン対戦** — ランダムマッチ。約20秒待っても人数が揃わなければ、空席をCPUが自動で埋めて開始（最低4人卓）。「待たずに始める」も可能
- **CPU対戦** — 相手の人数（1〜4人）を選んで即開始。サーバーを介さずブラウザ内だけで進行する

## 技術構成

- Next.js (App Router) + TypeScript。リアルタイム通信は使わず、ターン制＋ポーリングで進行
- ゲーム進行はすべてサーバー側（`lib/engine.ts` の状態機械）で処理。手札や伏せた札はクライアントに送られない
- CPUは手札の残り・公開済み札・宣言履歴から嘘の確率を見積もってダウトを判断（`lib/cpu.ts`）

## 開発

```sh
npm install
npm run dev
```

http://localhost:3000 で起動します。

## Vercel へのデプロイ

1. このリポジトリを Vercel にインポート（フレームワークは Next.js として自動検出）
2. そのままデプロイすれば動きます（無料の Hobby プランでOK）

### 本番でのルーム保存について

既定ではルーム状態をサーバープロセスのメモリに保存します。ローカルや小規模な利用では
そのまま動きますが、Vercel のサーバーレス環境ではインスタンスが分かれると別の卓が
見えなくなることがあります。安定運用したい場合は、無料枠のある
[Upstash Redis](https://vercel.com/marketplace/upstash) を Vercel Marketplace から接続し、
以下の環境変数を設定してください（設定すると自動で Redis 保存に切り替わります）:

- `UPSTASH_REDIS_REST_URL`
- `UPSTASH_REDIS_REST_TOKEN`
